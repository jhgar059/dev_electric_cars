Fase de Planeación y Diseño del Proyecto

1. Diagrama de Clases (Conceptual)

El diagrama de clases es una representación visual de la estructura de las clases del sistema, mostrando sus atributos, operaciones y las relaciones entre ellas. Para este proyecto, las clases principales reflejan las entidades de negocio:

  • AutoElectrico: Representa un vehículo eléctrico con sus características principales.
  • Carga: Detalla la dificultad y parámetros de carga para diferentes modelos.
  • Estacion: Describe las estaciones de carga disponibles.
  • Entidades Eliminadas: Clases espejo para mantener el historial de registros eliminados (AutoEliminado, CargaEliminada, EstacionEliminada).

Las relaciones son principalmente de composición (la carga y la estación de carga son entidades separadas con sus propios atributos) y de herencia (los modelos SQL heredan de `Base` y los modelos Pydantic de `BaseModel`).

Representación simplificada de las clases principales del sistema.

2. Modelo de Base de Datos (Relacional)

La base de datos se diseñó utilizando un modelo relacional para asegurar la integridad y la eficiencia en el almacenamiento y recuperación de datos. Se utilizan tablas separadas para cada entidad principal y tablas adicionales para los registros eliminados, permitiendo una gestión robusta del historial.

  • autos_electricos: (id, marca, modelo, anio, capacidad_bateria_kwh, autonomia_km, disponible, url_imagen)
  • dificultad_carga: (id, modelo, tipo_autonomia, autonomia_km, consumo_kwh_100km, tiempo_carga_horas, dificultad_carga, requiere_instalacion_domestica, url_imagen)
  • estaciones_carga: (id, nombre, ubicacion, tipo_conector, potencia_kw, num_conectores, acceso_publico, horario_apertura, coste_por_kwh, operador, url_imagen)
  • Tablas de Eliminados: autos_eliminados, dificultad_carga_eliminados, estaciones_eliminadas con estructuras espejo.
Esquema de las tablas y sus relaciones básicas.

3. Tecnologías Seleccionadas

La elección de las tecnologías se basó en la robustez, escalabilidad, facilidad de desarrollo y la popularidad en el ecosistema de desarrollo web moderno.

Python

Lenguaje de programación principal, conocido por su sintaxis clara y su vasto ecosistema de librerías.

FastAPI

Framework web moderno y rápido para construir APIs, con validación de datos automática con Pydantic.

PostgreSQL & SQLAlchemy

Base de datos relacional robusta y un ORM potente para interactuar con ella de forma programática.

HTML, CSS (Bootstrap), JavaScript

Tecnologías estándar para la creación de la interfaz de usuario, con Bootstrap para un diseño responsivo.

Git & GitHub

Sistema de control de versiones y plataforma de colaboración para la gestión del código fuente.

Render (Despliegue)

Plataforma en la nube para el despliegue continuo de la aplicación y la base de datos.
